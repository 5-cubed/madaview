# 2026-07-12

- Cut and published madaview's first release, v0.1.0, as a GitHub pre-release (`.context/adr/20260712-000652-cut-v0-1-0-release.md`). Added `prerelease: true` to the `release.yml` publish step, pushed the `v0.1.0` tag, and verified the full pipeline live: test, all 4 build-matrix targets (linux/amd64, windows/amd64, darwin/amd64, darwin/arm64), and publish all succeeded, producing https://github.com/5-cubed/madaview/releases/tag/v0.1.0 with all 4 binaries attached, marked pre-release, and correctly excluded from GitHub's "latest".
- Replaced the one-URL-one-file viewing model with an editor-style tabs + split-view workspace: multiple markdown files can be open as tabs (up to 2 panes side-by-side), tabs stay mounted and never re-fetch or lose scroll position, and the URL keeps tracking the focused pane's active tab for deep-linking/back-forward (`.context/adr/20260712-001324-tabs-and-split-view.md`).
- Added a client-side, instant markdown-rendering theme system: a Settings page `ThemePicker` (hover-to-preview, click-to-commit, persisted per-browser in localStorage) switches among 6 named palettes (GitHub Light, GitHub Dark, Kanagawa, Gruvbox, Solarized Light, One Dark) via a single `data-theme` attribute and CSS custom properties, recoloring headers, code blocks, tab strip, sidebar, header bar, and Settings itself with zero React re-render and zero server round-trip. Also fixed a real pre-existing bug as part of this work: Chroma-highlighted code blocks previously rendered as unstyled black text because no CSS ever defined their emitted token classes — now covered via ~8 attribute-prefix rules per theme (`.context/adr/20260712-221004-md-rendering-theme.md`).
- Fixed two contrast/hierarchy defects in the theme system: hand-tuned `--h1`..`--h6` in all 6 themes so no heading level is ever color-identical to `--text` or an adjacent level, and every level clears WCAG AA large-text contrast (3:1) against its theme's background; and wired the full set of `@tailwindcss/typography`'s `--tw-prose-*` CSS variables (body, bold, quotes, captions, code, list markers, table borders, `<hr>`, code-block background) to existing theme tokens, since only headings and links were previously wired — dark themes were rendering ordinary paragraph/list/blockquote/table text dark-on-dark. Added a reusable WCAG contrast-ratio helper (`e2e/lib/contrast.mjs`) and extended the `theme-switching` e2e scenario with heading-hierarchy and per-element WCAG checks (`.context/adr/20260712-233457-theme-contrast-hierarchy-fix.md`).
- Fixed a follow-up gap in the same theme fix: table header cells (`thead th`) render via Typography's `--tw-prose-headings` var too, not just `h1`-`h6`, and that var had been left unwired on the assumption that the existing `.prose h1`..`.prose h6` selectors already covered it — table headers were still rendering dark-on-dark. Wired `--tw-prose-headings` to `--text` and added a `tableHeader` representative check (4.5:1 WCAG gate) to `theme-switching`.
