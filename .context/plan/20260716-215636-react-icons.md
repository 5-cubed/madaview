# Plan: Add react-icons dependency for VSCode-style icons

**ADR:** `.context/adr/20260716-215527-react-icons.md`

> Note: the ADR's own Test-Loop Design states no e2e scenario applies —
> this is a manifest-only change with zero source-code behavior, so there
> is no RED to write a failing test against. The sequence below substitutes
> the ADR's Observability and Verification Criteria checks for the usual
> test-then-implement cycle, run in the same before/after order.

## Action Sequence
1. Edit `web/package.json`: add `"react-icons": "^5.7.0"` to `dependencies`.
2. Run `npm install` in `web/`, which updates `web/package-lock.json` with
   the new dependency and its transitive entries.
3. Verify: run `npm ls react-icons` in `web/` and confirm it reports
   `react-icons@5.7.0` with no `UNMET PEER DEPENDENCY` or `invalid` line
   (ADR Verification Criteria #1).
4. Verify: run `npm run build` in `web/` and confirm it succeeds, matching
   its pre-change baseline (no new errors, since nothing imports the
   package yet) (ADR Verification Criteria #2).
5. Verify: run `npm run lint` in `web/` and confirm it succeeds, matching
   its pre-change baseline (ADR Verification Criteria #2).
6. Verify: run `git diff web/package.json web/package-lock.json` and
   confirm the diff contains exactly one new dependency entry in
   `package.json` plus its transitive lock entries — nothing else changed
   (ADR Observability).
