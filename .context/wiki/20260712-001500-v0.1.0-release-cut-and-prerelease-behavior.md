# v0.1.0 release cut and GitHub "latest" behavior

Madaview's first real release, `v0.1.0`, was cut by pushing the tag to `origin`
(`.github/workflows/release.yml`). Confirmed live on the real repo:

- Setting only `prerelease: true` on the `softprops/action-gh-release@v2` step (no
  `make_latest` override) is sufficient to exclude a release from GitHub's "latest"
  computation. Verified via `gh api repos/5-cubed/madaview/releases/latest` returning
  `404` when `v0.1.0` is the only release in the repo and is marked prerelease — GitHub
  genuinely computes "latest" from the newest non-draft, non-prerelease release, with no
  extra flag needed.
- The full pipeline (`test` → `build` 4-target matrix → `publish`) works end-to-end on a
  real tag push: all 4 targets built, 3 were smoke-tested natively (linux/amd64,
  windows/amd64, darwin/arm64), and `darwin/amd64` (no native runner) built successfully
  with its smoke-test step skipped — the accepted Ambiguous Zone from the initial-setup
  ADR, and it still produces a working binary asset attached to the release.
